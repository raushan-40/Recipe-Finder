# Prompts Used During Development

This document records the AI prompts used while building the **Recipe Finder** project with the assistance of GitHub Copilot and ChatGPT.

---

# Project Overview

**Project:** Recipe Finder

**Objective:** Build a responsive web application that allows users to search for recipes using a public Recipe API and display detailed recipe information with proper loading states, validation, and error handling.

---

# Prompt 1 – Project Setup

```text
Create a Recipe Finder project using HTML, CSS, and Vanilla JavaScript. Keep the project responsive, beginner-friendly, and well-structured. Do not use any frameworks.
```

---

# Prompt 2 – HTML Structure

```text
Create the HTML structure for a Recipe Finder application with a search input, search button, recipe container, loading indicator, and error message section using semantic HTML.
```

---

# Prompt 3 – UI Design

```text
Design a modern dark-themed responsive UI using only CSS. Use cards, smooth spacing, rounded corners, hover effects, and a clean professional layout.
```

---

# Prompt 4 – Search Functionality

```text
Implement JavaScript to read the recipe name from the search input and start the search when the Search button is clicked or when the Enter key is pressed.
```

---

# Prompt 5 – API Integration

```text
Use Fetch API with async/await to retrieve recipe data from a public Recipe API. Parse the JSON response and handle possible request failures.
```

---

# Prompt 6 – Display Recipes

```text
Render recipe cards displaying the recipe image, name, category, cuisine, and a button to view complete recipe details.
```

---

# Prompt 7 – Recipe Details

```text
Display detailed recipe information including ingredients, cooking instructions, and external source or YouTube links if available.
```

---

# Prompt 8 – Loading State

```text
Show a loading spinner while the API request is in progress and hide it once the data has been loaded.
```

---

# Prompt 9 – Input Validation

```text
Validate the search input. If the input is empty, display a user-friendly message instead of sending an API request.
```

---

# Prompt 10 – Error Handling

```text
Handle invalid searches and API failures gracefully. Display a clean "No recipes found" message without breaking the application.
```

---

# Prompt 11 – Responsive Design

```text
Make the application fully responsive for desktop, tablet, and mobile devices while maintaining a clean layout.
```

---

# Prompt 12 – Accessibility

```text
Improve accessibility by adding proper labels, alt text, keyboard support, and ARIA attributes where appropriate.
```

---

# Prompt 13 – UI Improvements

```text
Improve spacing, typography, hover effects, animations, and overall user experience without changing the application's functionality.
```

---

# Prompt 14 – Code Refactoring

```text
Refactor the JavaScript into reusable functions. Keep the code modular, readable, and beginner-friendly.
```

---

# Prompt 15 – Bug Fixes

```text
Review the project for bugs related to search, API requests, rendering, responsiveness, and user interactions. Fix only the necessary code while preserving existing functionality.
```

---

# Prompt 16 – Final Code Review

```text
Review the entire project for code quality, responsiveness, accessibility, best practices, and performance. Suggest and implement only meaningful improvements.
```
